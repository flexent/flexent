# Governance

## Purpose

Flexent is an independent, frameworkless application toolkit maintained as an open-source project.

This document defines ownership, decision-making, and contribution boundaries so that organizations can adopt Flexent without ambiguity around project control or intellectual property.

## Ownership and Licensing

- Flexent source code is licensed under the MIT License.
- Copyright is held by Boris Okunskiy, unless explicitly stated otherwise in a specific file.
- The `Flexent` name and package namespace (`@flexent/*`) are maintained by the project maintainer.

## Maintainer Model

- The project is maintained by Boris Okunskiy as the primary maintainer.
- The maintainer is responsible for release decisions, roadmap direction, and final merge authority.
- Additional maintainers may be added explicitly by the primary maintainer.

## Contributions

- Contributions are accepted via pull requests.
- By submitting a contribution, contributors agree that their contribution is provided under the repository license (MIT).
- Contributors must only submit code they are authorized to contribute.
- Upstream changes to Flexent should remain general-purpose and reusable.

## Decision Principles

- Keep packages small, explicit, and composable.
- Prefer frameworkless and dependency-injection-friendly design.
- Optimize for long-term maintainability and low coupling.
